# Threat Hunting

## Overview

The Threat Hunting module in Wazuh was used to monitor and analyze security events generated by Linux endpoints connected to the SIEM environment.

The objective was to verify that Wazuh correctly detected user activity, authentication events, privilege escalation attempts, and host-based security events.

---

## Monitoring Scenario

During the laboratory exercise, several activities were performed to generate security events, including:

- SSH login attempts using invalid credentials
- User authentication
- Privilege escalation using `sudo`
- System service activity
- Host integrity monitoring
- Network port monitoring

The generated events were collected by the Wazuh Agent and analyzed through the Wazuh Dashboard.

---

## Security Events Observed

The following security events were detected during monitoring:

| Event | Severity | Description |
|--------|:--------:|-------------|
| PAM Login Session Opened | 3 | User authentication session started. |
| PAM Login Session Closed | 3 | User authentication session ended. |
| Successful sudo to ROOT | 3 | Privilege escalation using `sudo`. |
| Wazuh Agent Started | 3 | Endpoint agent successfully started. |
| Wazuh Agent Stopped | 3 | Endpoint agent stopped. |
| Host-based Anomaly Detection (Rootcheck) | 7 | Rootcheck detected potential security anomalies. |
| Listened Ports Status Changed | 7 | Network port status changed. |
| SSH Hardening Assessment | 7 | Security configuration assessment identified SSH hardening recommendations. |
| New User Added *(if detected)* | 8 | User account creation detected by Wazuh. |

## Severity Scale

| Level | Interpretation |
|:-----:|----------------|
| 3 | Low – Informational security event requiring review. |
| 7 | High – Potentially suspicious activity requiring investigation. |
| 8 | High – Significant security event requiring immediate attention. |

---

## Event Analysis

The generated alerts confirmed that Wazuh successfully monitored authentication events, privilege escalation, host integrity, and network activity.

Several security events were automatically classified according to their severity levels, allowing faster identification of potentially suspicious activity.

Examples of detected activity included:

- SSH authentication attempts
- Administrator privilege usage
- Host integrity monitoring (Rootcheck)
- Network port changes
- Endpoint status monitoring

---

## Detection Techniques

The monitoring process relied on multiple Wazuh detection mechanisms, including:

| Technique | Purpose |
|-----------|---------|
| Authentication Monitoring | Detect user login activity and authentication events. |
| Rootcheck | Identify potential rootkits and host integrity anomalies. |
| Security Configuration Assessment (SCA) | Evaluate system configuration against security best practices. |
| Netstat Monitoring | Detect changes in network listening ports. |
| Agent Monitoring | Monitor endpoint connectivity and agent status. |

These mechanisms provided continuous visibility into endpoint activity and system configuration.

## Results

The laboratory successfully demonstrated the capabilities of the Wazuh Threat Hunting module in detecting and correlating security events generated by monitored Linux endpoints.

The platform provided centralized visibility into authentication activity, privilege escalation, host integrity monitoring, and system configuration changes.

The collected events confirmed that the deployed SIEM environment was capable of supporting basic security monitoring and incident investigation workflows.
