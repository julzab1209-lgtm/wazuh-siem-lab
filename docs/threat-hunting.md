# Threat Hunting

## Overview

The Threat Hunting module in Wazuh was used to monitor and analyze security events generated by Linux endpoints connected to the SIEM environment.

The objective was to verify that Wazuh correctly detected user activity, authentication events, privilege escalation attempts, and host-based security events.

---

## Monitoring Scenario

During the laboratory exercise, several activities were performed to generate security events, including:

- SSH login attempts using invalid credentials
- User authentication
- Privilege escalation using `sudo`
- System service activity
- Host integrity monitoring
- Network port monitoring

The generated events were collected by the Wazuh Agent and analyzed through the Wazuh Dashboard.

---

## Security Events Observed

The following security events were detected during monitoring:

| Event | Description |
|--------|-------------|
| PAM Login Session Opened | User authentication session started |
| PAM Login Session Closed | User authentication session ended |
| Successful sudo to ROOT | Privilege escalation using sudo |
| Host-based Anomaly Detection (Rootcheck) | Detection of potential security anomalies |
| Wazuh Agent Started | Endpoint agent successfully started |
| Wazuh Agent Stopped | Endpoint agent stopped |
| Listened Ports Status Changed | Network port status changed |
| SSH Hardening Checks | Security configuration recommendations for SSH |

---

## Event Analysis

The generated alerts confirmed that Wazuh successfully monitored authentication events, privilege escalation, host integrity, and network activity.

Several security events were automatically classified according to their severity levels, allowing faster identification of potentially suspicious activity.

Examples of detected activity included:

- SSH authentication attempts
- Administrator privilege usage
- Host integrity monitoring (Rootcheck)
- Network port changes
- Endpoint status monitoring

---

## Detection Techniques

The monitoring process relied on multiple Wazuh detection mechanisms, including:

- Authentication log monitoring
- Rootcheck
- Security Configuration Assessment (SCA)
- Network monitoring (Netstat)
- Linux audit logs

These mechanisms provided continuous visibility into endpoint activity and system configuration.

---

## Results

The Threat Hunting module successfully collected and categorized security events generated during the laboratory exercises.

The collected alerts demonstrated that Wazuh can effectively support:

- centralized security monitoring
- incident investigation
- endpoint visibility
- security event correlation
- early detection of suspicious activity
