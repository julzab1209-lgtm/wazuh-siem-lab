# Lessons Learned

## Overview

The Wazuh SIEM laboratory project provided practical experience in deploying, configuring, and operating a Security Information and Event Management (SIEM) platform within a virtual environment.

Throughout the project, I gained a better understanding of security monitoring workflows and the importance of centralized log analysis in detecting potential security incidents.

---

## Key Lessons

- Deploying a SIEM platform requires proper planning of the virtual environment and system configuration.
- Endpoint monitoring provides valuable visibility into authentication events, privilege escalation, and system activity.
- Threat Hunting enables proactive investigation of security events rather than relying solely on automated alerts.
- Mapping events to the MITRE ATT&CK framework helps classify attacker techniques and better understand security incidents.
- Well-structured technical documentation is an important part of every cybersecurity project.

---

## Challenges

During the implementation, several technical challenges were encountered, including:

- configuring communication between the Wazuh Manager and monitored endpoint,
- verifying agent connectivity,
- interpreting generated security events,
- organizing technical documentation in a clear and structured manner.

Overcoming these challenges improved both technical troubleshooting skills and understanding of SIEM deployment processes.
